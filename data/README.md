# Datasets

Two hand-curated relation sets drive every experiment. All relations are audited; `audit_confidence`
and `source_or_rationale` record provenance per item.

## `causal_400.json` — 400 directed causal relations

`{"relations": [ {...}, ... ]}`. **300 positive-polarity** + **100 negative-polarity**.

| Field | Meaning |
|-------|---------|
| `id`, `pair_id` | unique id / grouping id (the bootstrap and GroupKFold **resampling unit**) |
| `x_text`, `y_text` | cause and effect surface strings (e.g. `smoking` → `lung cancer`) |
| `x_low`, `x_moderate`, `x_high` | graded cause-intensity phrasings for the **dose** axis (e.g. `light` / `moderate` / `heavy smoking`) |
| `polarity` | **`positive`** = more X → more Y (used in Stages 1–9). **`negative`** = more X → *less* Y (protective; held out for the polarity-dissociation test) |
| `true_direction`, `true_graph` | ground-truth arrow / graph string |
| `relationship_type`, `domain`, `mechanism` | metadata: edge type, topic domain, brief mechanism |
| `keep_for_headline`, `audit_confidence`, `source_or_rationale` | curation flags and provenance |

**Polarity matters for the experiment.** For positive relations, intensifying the cause should *raise*
confidence in "X causes Y". For negative relations (e.g. *more exercise → less heart-disease risk*),
intensifying the cause should *lower* it — the sign-flip is the discriminating test that separates a true
causal-dose representation from mere intensity-word salience. The notebook reads `polarity` as
**authoritative** (it does not re-derive it) and holds the 100 negatives in `CAUSAL_NEG_ITEMS`.

```json
{
  "id": "causal_001", "pair_id": "causal_001",
  "x_text": "smoking", "y_text": "lung cancer",
  "x_low": "light smoking", "x_moderate": "moderate smoking", "x_high": "heavy smoking",
  "polarity": "positive", "true_direction": "x->y", "domain": "health"
}
```

## `spurious_forks_300.json` — 300 common-cause "fork" pairs

The spurious control: X and Y are both downstream of a confounder Z, with **no direct edge** — so the
correct continuation is "not directly causally related". Built from the audited
`fork_spurious_candidates_300.csv`.

| Field | Meaning |
|-------|---------|
| `id`, `pair_id` | unique / grouping id |
| `x_text`, `y_text` | the two correlated-but-not-causal variables |
| `confounder_z` | the common cause Z (e.g. `hot weather`) |
| `true_direction` | always `none` |
| `spurious_subtype`, `why_no_direct_edge`, `domain` | structure + justification |
| `keep_for_headline`, `audit_confidence`, `source_or_rationale` | curation flags |

```json
{
  "id": "fork_001", "pair_id": "fork_001",
  "x_text": "ice cream sales", "y_text": "drowning incidents",
  "confounder_z": "hot weather", "true_direction": "none"
}
```

## How they're combined (notebook §2)

The loader builds three behavioural categories from these files:
**causal** (positive relations, forward arrow), **anti-causal** (the same relations reversed), and
**spurious** (the forks). Anti-causal items are generated by swapping `x_text`/`y_text`, so a model that
merely tracks first-mention word order cannot separate them from causal — which is exactly what the
Stage 5 word-order baseline tests.
